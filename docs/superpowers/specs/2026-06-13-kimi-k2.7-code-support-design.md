# Add `kimi-k2.7-code` support to the `kimi-coding` provider

## Context

The `code-switch` tool already has a built-in provider called `kimi-coding` that targets the Kimi Coding API. Currently that preset only exposes a single model: `kimi-for-coding`.

Kimi now offers a new model, `kimi-k2.7-code`, which should become the recommended default for the provider while keeping the previous model available as a fallback.

## Goal

Make `kimi-k2.7-code` the default model for the `kimi-coding` provider and keep `kimi-for-coding` as a selectable alternative.

## Design

### Provider preset update

Edit the `kimi-coding` entry in `presets.go`:

- Set `Model` to `kimi-k2.7-code`.
- Update `Models` to `[]string{"kimi-k2.7-code", "kimi-for-coding"}` (new default first).
- Update `Haiku`, `Sonnet`, `Opus`, and `Subagent` to `kimi-k2.7-code`.
- Keep all other fields unchanged:
  - `Name`: "Kimi Coding"
  - `BaseURL`: `https://api.kimi.com/coding/`
  - `AuthEnv`: `ANTHROPIC_API_KEY`
  - `Website` / `APIKeyURL`: `https://platform.kimi.com`
  - `ExtraEnv`: `API_TIMEOUT_MS=3000000`, `CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC=1`

### Agent impact

The `kimi-coding` preset is consumed by both Claude Code and Codex:

- **Claude Code** uses `resolveProviderPreset` / `resolveSwitchPreset` directly.
- **Codex** uses `codexKimiCodingPreset()` in `agent.go`, which copies the preset, changes the base URL to `https://api.kimi.com/coding/v1`, sets `AuthEnv` to `KIMI_API_KEY`, and enables `ForceModelTiers`.

Because Codex derives from the same preset, updating the default model in `presets.go` automatically applies to Codex as well. No additional agent-specific changes are required.

### Detection / aliases

The existing alias mapping already resolves `kimi`, `kimi.com`, and `api.kimi.com` to `kimi-coding`. No alias or detection changes are needed.

### Tests

Existing tests only verify that `kimi-coding` appears in provider lists; they do not assert the specific default model. After the change, run the full test suite with `go test ./...` to confirm no regressions.

### Documentation

No README or shell completion changes are required because `kimi-coding` is already a known preset. The interactive TUI and `cs list --verbose` will automatically show the updated model list.

## Scope

In scope:

- Updating the `kimi-coding` preset in `presets.go`.
- Running the test suite to verify no regressions.

Out of scope:

- Adding new aliases or a separate provider.
- Changing Codex-specific auth/base URL behavior.
- Updating OpenCode Go unsupported-model list (no change needed; `kimi-k2.7-code` is not listed there).

## Acceptance criteria

- `cs switch kimi-coding` selects `kimi-k2.7-code` as the model.
- `cs list --verbose` shows `kimi-k2.7-code` first, followed by `kimi-for-coding`.
- `cs switch kimi-coding --model kimi-for-coding` still works.
- `go test ./...` passes.
